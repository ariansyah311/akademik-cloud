Arsitektur Cloud & Backup

Penjelasan arsitektur dan pengelolaan backup data

Arsitektur Cloud yang Digunakan

Aplikasi ini dirancang untuk dihosting pada layanan Platform-as-a-Service (PaaS) seperti Railway, dengan arsitektur sebagai berikut:

  • Frontend: HTML, CSS, JavaScript (vanilla), disajikan secara statis oleh server Express.
  • Backend: Node.js + Express.js menangani REST API (autentikasi, CRUD, statistik).
  • Database: Disimulasikan dengan file JSON (data/db.json) yang tersimpan pada volume server cloud. Pada implementasi nyata dapat digantikan dengan layanan seperti Railway PostgreSQL, AWS RDS, MongoDB Atlas, dsb.
  • Autentikasi: JSON Web Token (JWT) dengan password yang di-hash menggunakan bcrypt — mengimplementasikan authentication (siapa yang login) dan authorization berbasis role (admin, dosen, mahasiswa).
  • Hosting: Seluruh aplikasi (frontend + backend) digabung dalam satu service Node.js yang dideploy ke Railway melalui koneksi GitHub repository atau Railway CLI, sehingga dapat diakses secara online melalui domain publik (*.up.railway.app).

Alur Request: Client (Browser) → HTTPS → Railway Load Balancer → Express Server (Node.js) → File-based Database → Response JSON.

Backup Data

Sistem mendukung dua mode backup:

  • Backup Otomatis: Dijalankan terjadwal setiap hari pukul 02:00 melalui cron job pada server (lihat server.js), menyalin db.json ke folder data/backups/.
  • Backup Manual: Admin dapat memicu backup kapan saja melalui tombol di bawah ini.
